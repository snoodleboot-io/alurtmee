Meet Alurtmee

A companion, not a dashboard.

Alurtmee is the name of the mascot, too — the glowing orb that keeps watch. The rest of the time it stays quiet and out of the way.

Always current

Polls in the background and resumes automatically next launch — no re-entry, no babysitting.

Two-pane feed

Your PRs on the left; click one to see its full detail — reviews, CI dots, security edge — on the right.

GPU-rendered, native

A real desktop app built in Rust — fast, lightweight, and crisp on your display.

Getting connected

Set it up once. Then forget it's there.

Add a token

Create a GitHub personal access token with read access and paste it into Settings. It's saved straight to your system keychain — add as many as you need (personal, work, org).

Pick your repos

Tick the repositories you'd like to keep an eye on. The repos all your tokens can see combine into one list — a repo visible to more than one token still appears once.

Get on with your day

Head back to the feed and it fills in. From here it watches quietly, reconnecting and resuming on every launch — and nudges you only when a build breaks or drags.

Your data stays yours

Private by design.

Alurtmee asks for read access, holds your token where the OS keeps secrets, and keeps everything else in a small local database on your machine.

  • Token in the keychain only. Via the system Secret Service — never written to the database, config, or logs.
  • Local cache, local choices. Your repo selection, theme and preferences live in a small DB under your user data directory.
  • Verifiable builds. Every release ships a SHA256SUMS file and a signed build-provenance attestation.
  • Open source. Apache-2.0. Read every line, build it yourself, send a patch.
verify your download
# confirm the binary is intact…
$ sha256sum -c SHA256SUMS

# …and that this repo's CI built it
$ gh attestation verify Alurtmee.AppImage \
    --repo snoodleboot-io/alurtmee

# prefer no install? try it with sample data:
$ ALURTMEE_DEMO=1 ./Alurtmee.AppImage
